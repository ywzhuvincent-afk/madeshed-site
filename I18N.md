# 中英文双语规约（强制 · 任何改动都要遵守）

> 铁律：**中文用中文、英文用英文，各自地道；任何一次改动都必须把两种语言同步改好，绝不允许一种语言的界面里露出另一种语言的文字**（八字干支等专有符号除外，见下）。
>
> 违反这条最典型的事故就是「英文用户点购买，进去还是中文名/中文按钮」——本文件就是为杜绝这类问题而立。

---

## 0. 一句话检查清单（每次提交前对照）
任何涉及**用户可见文字、新功能、新商品、新页面、新接口返回文案**的改动，必须逐条过：

1. [ ] 中文文案 ✅ 有；英文文案 ✅ 有；两者**各自母语地道**（不是把中文机翻硬塞，也不是中英混排）。
2. [ ] 新增的**动态值/命理值**（分数、标签、十神、状态句…）：中文渲染路径和英文渲染路径**都**覆盖到（要么渲染函数本身按 `localeIsEn()` 分支，要么在 EN 覆盖层里补上）。
3. [ ] 新增/改动**Stripe 商品或结账**：`checkout.js` 里该商品有 `label`(中) 和 `labelEn`(英)，会话传了 `locale`，结账页按语言显示各自商品名。
4. [ ] **绝不**让某语言界面残留另一语言（唯一豁免：八字**干支/天干地支**字符——英文八字工具通用做法是保留汉字，如 `甲申`、`申子辰`）。
5. [ ] **中英文各自在浏览器里走一遍**验证（切到 EN、切到 ZH 各看一次），不是只测一种。
6. [ ] EN 覆盖层**只做翻译，不得用样例/罐头数据顶掉真实值**（历史事故：英文态用写死的 `12 次/67%` 覆盖真实统计）。

## 1. 双语是怎么实现的（架构）
本站没有 key-based i18n 框架，是**「渲染后按语言处理」**的模式，务必理解清楚再动：

- `localeIsEn()` / `chartLocaleIsEn()`：判断当前语言。语言存在 `localStorage` 的 `madeshed_locale_v1`，登录后可被账号 `profile.locale` 覆盖。
- **静态文案**：`EN_COPY`（`index.html`）+ `localizeEnglishWholeSite` / `setLocaleText` —— 按 CSS selector 把中文替换成英文。加静态文案时**两边都要加**：HTML 里写中文，`EN_COPY`/`localize*` 里补对应英文。
- **动态渲染的命理值**：分两种正确做法，二选一——
  - (a) 渲染函数**本身**按 `localeIsEn()` 出对应语言（首选，最不容易漏）；
  - (b) 渲染中文，再由 `localizeEnglishDynamicWidgets` 等 EN 覆盖层翻译。**若走 (b)，切换语言时必须能重渲染**（否则英文态点开新内容还是中文）。
- **chart-full.html 流日明细**：命理「值」的英译集中在 `ld*` 系列（`ldBandEn / ldPosEn / ldRoleEn / ldTenShort / ldCLabelEn / ldSpecialEn / ldInterEn`）。新增任何流日/命理展示字段，英译要在这里补。
- **结账（Stripe）**：见下一节，这是最容易漏的地方。

## 2. 结账 / Stripe 的双语（重点，别再翻车）
Stripe 结账页有**两层语言**，都要管：

1. **Stripe 界面本身**（Pay 按钮、字段名、邮箱/卡号标签…）：由结账会话的 `locale` 参数决定。
   - 前端 4 个购买入口（`beginCreditCheckout / beginMembershipCheckout / beginReportCheckout / beginFortuneReportCheckout`）在 POST body 里带 `locale:(localeIsEn()?'en':'zh')`。
   - 后端 `api/checkout.js` 的 `checkoutLocale(req)` 读它，`params.set('locale', locale)`。
2. **商品名/描述**：**不能**只用固定 `price` ID（那样名字永远是 Stripe 里那一份中文）。用 `setLocalizedLineItem()`：金额/币种/订阅周期仍取自已配置的 `price`（价格权威、不动），**只把展示名换成 `label`(中) 或 `labelEn`(英)**。
   - 因此**每个商品在 `checkout.js` 里必须同时有 `label` 和 `labelEn`**。加新商品、改商品名，两者一起改。
3. Stripe 后台里那些持久化的「商品名」（`business_profile`、Product catalog 里的名字）只影响后台/收据登记，**结账页展示以 `price_data` 的 `labelEn/label` 为准**。

> 若以后要英文用户看到英文**收据/邮件**，Stripe 的收据模板本身也需在 Stripe 后台配英文，或按 locale 分模板——目前 `locale` 已能覆盖结账页与大部分自动邮件语言。

## 3. 只测一种语言 = 没测
每次改动**必须两种语言各在浏览器里过一遍**：切到英文点一遍、切到中文点一遍。历史上所有「英文残留中文」的 bug 都是因为只测了中文。

## 4. 已知豁免
- **八字干支**（天干地支、干支组合，如 `甲申`、`申子辰`、`癸亥辛酉甲子戊辰`）：英文态**保留汉字**，这是英文八字工具的通行做法，不算「露出中文」。其余一切（标签、说明、按钮、商品名、状态句）英文态必须是英文。
